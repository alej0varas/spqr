renderCityNames(self)

source code 

Routine renders all the images for the city names, and places them in the city instances. Must be done in main init routine before any map rendering is performed

addWindow(self, window)

source code 

Call to add a window to the gui window list. It always goes on the top of the window pile, and thus if modal traps all user input. Returns the index number of the window in the list, so you can amend the window afterwards

killIndexedWindow(self, index)

source code 

Remove window. Call with index number of window. Redraws gui as well

killTopWindow(self)

source code 

Remove top window. Redraws gui as well

updateGUI(self)

source code 

Redraws entire screen. Should avoid calling this really and use dirty rectangles technique. Having said that, it's not actually that slow either

updateMap(self)

source code 

Updates (i.e. redraws) map to main screen

updateUnits(self)

source code 

Unit is to be called when a unit is placed or removed from the map. Always returns True

centreMap(self, xpos, ypos)

source code 

Centre map to the given co-ords, or at least as close as you we get. DOES NOT update the screen for you

getHexOnscreen(self, xpos, ypos)

source code 

Call routine with x and y co-ords of a hex on the board. It will attempt to return the x and y co-ords of a pixel on the screen that is in that hex (or else returns -1,-1)

normalizeScrollArea(self)

source code 

Checks co-ords after scrolling the map to make sure they are not out of range. Resets them if needed

checkInputs(self)

source code 

checkInputs() is called on a loop whilst the game is waiting for user input (i.e. most of the time). It doens't actually do anything with the input except pass the event along to somewhere else, so think of it more like a sorting office

checkButtonHighlights(self, x, y)

source code 

Check all of the buttons inside the top-layer window to see if any need to be highlighted. Returns True if anything on the screen needed to be updated

checkScrollArea(self, x, y)

source code 

If the map is scrolled by mouse on the outside, then call this routine with the x and y mouse co-ords. Returns True if the screen (and thus the map) were updated

updateInfoBox(self, x, y)

source code 

Updates information in bottom box, dependant on users click over the map. Call with x and y, being the click on the map in screen co-ords

drawHexUnitContents(self, x, y)

source code 

Draws and updates the 4 unit details to the left of the mini map. Call with x and y pos of hex

drawUnitInfo(self, index)

source code 

Call with the unit index number you want to display. Ultimatly this will show all the unit details you need to see

drawCityInfo(self, index)

source code 

Call with the city index number you want to display. Works in almost the same way as drawUnitInfo()

fitText(self, text, x, y, fnt)

source code 

Call with the text, the x and y size of the area to test against, and the font. Returns false if it couldn't be done, otherwise returns true and the image is in the gui spare image

renderPixelMap(self)

source code 

Routine generates map we actually display to the screen, having rendered cities, units, roads etc. The current ordering of the blits, from first to last, is: Back map, Cities, City names, Units

renderGameTurn(self)

source code 

Routine draws image for the game turn counter. Call at the start of every turn, just before updating the screen

flashUnit(self)

source code 

Flashes current highlighted unit. Returns True if the screen was updated, false otherwise

unitFlashAndOff(self)

source code 

Call to turn off unit flashing. Also draws unit back to screen if it is currently not on

unitFlashOn(self)

source code 

Call to turn unit flashing back on. Returns False if this didn't happen for some reason

prepareMove(self)

source code 

Routine to redraw all units EXCEPT the current highlight one onto a new back map. Returns the x,y postion of the current unit on the BACK_MAP screen (or at least, where it should be)

checkMoveBounds(self, rect)

source code 

To ensure that the whole move area gfx fits on the screen, we move the map if that's needed. Cheap but effective, since we have to display the hex the unit goes on afterwards anyway

animateUnitMove(self, xpos, ypos, direction, unit, battle)

source code 

This is the routine that animates a unit move. Call with the xpos and ypos of the unit, a direction flag to indicate movement, the unit to move (or -1 to use current highlighted unit), and finally a boolean to indicate wether battles should happen (or the unit stops dead) Returns True if the move took place
