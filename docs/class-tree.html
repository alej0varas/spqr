Class Hierarchy

  • object: The most base type
    • scripts.spqr_window.CButtonDetails
    • scripts.spqr_widgets.CCallbacks: Simple class holding callbacks for widgets.
    • scripts.maps.spqr_city.CCity: Defines a city in SPQR
    • scripts.spqr_gui.CDirtyRect
    • scripts.spqr_gui.CGFXEngine
    • scripts.spqr_data.CInfo
    • scripts.spqr_keys.CKeyboard: CKeyboard holds a list of keyboard events to respond to and the functions that they should call (defined in the normal SEVENT) Note: You can set one key to have more than one function in fact this is normal behaviour in a modal window if the window is modal, then the last one in the list will get called, if not then the first will get called.
    • scripts.spqr_keys.CKeypress: Class defines a simple keypress, as stores in the keyboard handling code
    • scripts.maps.spqr_map.CMap
    • scripts.spqr_menu.CMenu: Class holds all of the parent menus - thus there is only ever one instance of this class in our code
    • scripts.spqr_menu.CMenuChild: Class defines an item in a drop-down menu
    • scripts.spqr_menu.CMenuParent: Class defines a parent menu (something like the 'File' part of a normal drop-down menu)
    • scripts.player.spqr_player.CPlayer
    • scripts.maps.spqr_map.CRegion
    • spqr.CSPQR
    • scripts.spqr_sound.CSound: Class stores and sets music parameters
    • scripts.units.spqr_unit.CUnit: Normally the calling functions read q and m from a file, hence the odd maths when it calculates morale and quality
    • scripts.spqr_widgets.CWidget: Base class for widgets: All other widgets should build on this one
      • scripts.spqr_widgets.CButton: Init routine to create a button widget.
      • scripts.spqr_widgets.CCheckBox: Checkbox widget
      • scripts.spqr_widgets.CImage: Image class states and stores details for a simple image
      • scripts.spqr_widgets.CItemList: Call the init routine with the following parameters: first a list of lists, defined in the following way: The first list lets the routine know wether each element of the rest of the lists is text (True) or an image (False) The second list gives the text headers for the columns Of the rest of the lists, each seperate one contains all the elements needed to draw 1 column of the ScrollArea list Then comes an array of routines to sort the data Next comes a list of id numbers, used when sorting the data Finally, the last param tells you what height you would like the element to take up (including the header height)
      • scripts.spqr_widgets.CLabel: Label class stores details for a simple label
      • scripts.spqr_widgets.COptionMenu: OptionMenu widget lets user choose from a drop-down menu.
      • scripts.spqr_widgets.CScrollArea: ScrollArea class holds details for a ScrollArea Contains a graphical area that has a scroll bar on it's RHS Call with x/y positon, width and height and image *NOTE*: in the x size, you must allow for the fact that this widget will add a scrollbar on the rhs of a given size! height of widget *MUST BE* >61 pixels or funny gfx effects will occur border area, if true, adds 1 pixel on the lhs and 2 on the y axis
      • scripts.spqr_widgets.CSeperator: Seperator class states and stores details for a seperator
      • scripts.spqr_widgets.CSlider: Slider class states and stores details for a slider Call with following parameters: x/y pos of widget in window width - width of entire widget, start - value on lhs of widget end - value on rhs of widget, initial - initial value
      • scripts.spqr_widgets.CText: Text class stores details for a simple text widget and handles when clicked the keyboard input.
    • scripts.spqr_window.CWindow: Base window class Call with x,y - position width,height, title - text on top of window, draw - wether to place on screen or not
    • scripts.maps.spqr_map.Position
    • scripts.units.spqr_unit.UnitStats
