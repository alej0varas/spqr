 17  # SPQR parser for game save files, scenarios etc... 
 18  # look at SPQR documentation for full details, but the basic parts are: 
 19  #       a NAME is ^[0-9A-Za-z_]+$, and we convert the _'s to spaces 
 20  #       a NUMBER is [0-9]+ 
 21  #       a GFX is ^IMG_[0-9A-Za-z_]$, and we DON't convert the _'s 
 22  #   a PTYPE is ^(Human|Computer)$ 
 23  # There are (at the moment) 3 items to check for: 
 24  # firstly we have the players: 
 25  #                       player NAME 
 26  #                               (NAME) 
 27  #                               (NAME) 
 28  #                               (PLAYER_TYPE) 
 29  # a city: 
 30  #               city NAME 
 31  #                               (NUMBER,NUMBER) 
 32  #                               (GFX) 
 33  #       (NAME) 
 34  # a unit: 
 35  #               unit NAME 
 36  #                       (NUMBER,NUMBER) 
 37  #                       (GFX) 
 38  #     (NAME) 
 39  #                       (NUMBER,NUMBER,NUMBER) 
 40  # and a person: 
 41  #               person NAME 
 42  #                       (NATIONALITY) 
 43  #                       (AGE,SEX) 
 44  #                       (POPULOUS,LIBERALITY,REPUBLICAN) 
 45  #                       (BIRTHPLACE,BIRTHDATE) 
 46  #                       (UNITS:[NONE|X,X,X) 
 47  #                       (CITIES:[NONE|X,X,X) 
 48  # for example, a city might be: 
 49  #       city Rome 
 50  #       (20,31) 
 51  #       (IMG_ROME) 
 52  # Finally, each of the names given in the player 2nd line MUST be a  
 53  # city defined later 
 54  # a line starting with a # is a comment... 
 55   
 56  # Sometimes I pass back some non-obvious values (for example, the string to get  
 57  # the gfx name returns a STRING "False" if there was an error). This is for 
 58  # later, when we will make a 'force' option for bad or corrupt files.
